# YOLO 标签类别 ID 批量修改工具

## 功能简介
批量修改 YOLO 格式标注文件（`.txt`）中第一列的类别 ID（class_id）。  
支持：
- 仅修改指定旧 ID 的类别
- 强制将所有类别修改为同一 ID
- 自动创建备份（`.bak` 文件），防止误操作

## 命令行参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--label-dir` | 路径 | ✅ | 存放 `.txt` 标签文件的目录 |
| `--old-id` | 整数 | ❌ | 要修改的旧 ID；默认 `-1` 表示修改**所有**类别 ID |
| `--new-id` | 整数 | ✅ | 新的类别 ID（目标 ID） |
| `--no-backup` | 标志 | ❌ | 若指定，则不创建备份文件（`.bak`）；默认会备份 |

## 使用示例

### 1. 将标签目录下所有类别 2 改为类别 1，并自动备份
```bash
python change_id.py --label-dir /path/to/labels --old-id 2 --new-id 1
```

# YOLO 标注文件清理工具

## 功能简介
批量清理 YOLO 格式标注文件（`.txt`），支持：
- 删除指定类别的所有标注（如去除类别 `3`）
- 删除完全相同的重复标注（类别 + 坐标完全相同）
- 可选删除同一类别的多余标注（每个类别仅保留第一个出现的标注，即使坐标不同）
- 详细的统计报告（处理文件数、移除标注数、各类别移除数量等）

## 函数参数

```python
clean_yolo_txt_files_advanced(

    remove_class_ids=None,         # 要移除的类别 ID 列表，默认 [3]
    remove_duplicate_classes=False # 是否只保留每个类别的第一个标注
)
```

# YOLO 数据集自动化清理工具（同时处理图像和txt文件）

## 功能概述
该脚本用于自动清理 YOLO 格式数据集中的低质量或无效样本，可检测并删除以下四类问题：

- **损坏图像**：无法读取、尺寸为零、全黑或通道异常的图像
- **模糊图像**：基于拉普拉斯方差判断，低于阈值的视为模糊
- **森林火灾场景**：通过颜色分析与标签内容检测森林火灾图像（绿色背景 + 火焰/烟雾）
- **重复图像**：基于直方图特征和余弦相似度，删除高度相似的重复图像

所有被删除的图像和对应标签文件会先备份到指定目录，并生成详细的清理日志和统计报告。

## 工作原理

1. **目录结构验证**  
   要求数据集根目录下包含 `images/` 和 `labels/` 两个子目录，其中存放图像文件和对应的 `.txt` 标签文件（文件名相同，扩展名不同）。

2. **图像-标签配对**  
   扫描 `images/` 下常见格式（`.jpg`, `.png`, `.bmp` 等）的图像，查找 `labels/` 中同名的 `.txt` 文件，形成配对列表。

3. **四步检测流程**（顺序执行，每步在前一步保留的基础上进行）  
   - **损坏检测**：使用 OpenCV 读取图像，检查 `None`、尺寸、通道数、全黑等异常。  
   - **模糊检测**：将图像转为灰度图，计算拉普拉斯方差，低于 `blur_threshold` 的视为模糊。  
   - **森林火灾检测**：  
     - 在 HSV 空间检测绿色区域（森林）占比 > 30%。  
     - 检测红色/橙色区域（火焰）和低饱和度灰白色区域（烟雾）。  
     - 同时读取标签文件，检查是否有火灾/烟雾类别（默认 class_id 0=fire, 1=smoke）。  
     满足森林且（有火焰/烟雾或标签含火/烟）则判定为森林火灾场景。  
   - **重复检测**：对剩余图像提取 HSV 颜色直方图作为特征，使用最近邻算法计算余弦距离，删除相似度高于 `threshold` 的重复图像（保留索引较小的）。

4. **安全删除与备份**  
   所有被判定为无效的样本（图像 + 标签）会被移动到备份目录（默认在数据集根目录下创建 `backup_clean_时间戳` 文件夹），并在内存中记录删除日志。

5. **生成报告**  
   清理结束后，在数据集根目录下生成：
   - `cleaning_log.json`：包含完整删除记录、统计信息和备份路径。
   - `cleaning_summary.txt`：人类可读的清理报告。

## 命令行参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--data_dir` | 路径 | ✅ | 数据集根目录（必须包含 `images/` 和 `labels/` 子目录） |
| `--backup_dir` | 路径 | ❌ | 备份目录，默认为 `data_dir/backup_clean_时间戳` |
| `--threshold` | 浮点数 | ❌ | 图像相似度阈值，范围 0-1，默认 0.9。越高则删除越少（更严格） |
| `--blur_threshold` | 浮点数 | ❌ | 模糊检测阈值，默认 100.0。越小越容易判定为模糊（更严格） |
| `--dry_run` | 标志 | ❌ | 模拟运行：不实际删除文件，仅生成报告和备份到临时目录 |
| `--config` | 路径 | ❌ | 配置文件（JSON 格式），可预先设置参数（暂未实现完全覆盖） |
